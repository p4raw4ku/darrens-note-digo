# DIGO_EXE — 總覽

> 一款**兩人對戰回合制遊戲**的設計工作區(**數位、之後由 Claude 寫成程式**)。大方向:把《洛克人EXE》的格子晶片戰改寫成回合制,並借《聖劍同盟》的「卡片自帶移動力」——每張晶片同時具備移動與能力兩種性質,單次使用只能取其一。

## 這是一款什麼遊戲

雙方各操縱一名**領航員**,站在自己的九宮格上輪流行動,把對方 HP 打到 0 就贏。教學三句話:

1. 每回合的**行動有限**——移動、用晶片、用領航員的能力或攻擊,做了這件就做不了那件;
2. 每張晶片**二選一**——當移動用、或當能力用,用了這面就放棄那面;
3. 出手之前**先猜對方**——他下一步要走位、要打、還是在鋪什麼。

這三句話背後就是設計的憲法——**核心樂趣三支柱:選擇、資源分配、猜想對方與競爭**。之後的每一個機制、每一格數值,最後都要回答同一個問題:「你服務哪一支柱、有沒有傷到另外兩支?」

## 設計現在走到哪裡

這份設計經歷過一次完整的 v1 規則稿(2026-06,多 agent 產出、對抗測試修過 32 個破綻),之後 user 四點設計反思、再加一輪 22 條機制審查,把 v1 稿逼到了重整點。**2026-07-03 框架重整**:設計基準改為「**確立項與發想分離**」——

- **確立的**只有骨架:雙九宮格、輪流行動、領航員(能力/自攻/三防禦)、晶片二選一、傷害乘以次數+範圍、HP 歸零獲勝。收在 [[0_核心確立項]](唯一規則權威,每個領域一個分域檔,隨拍板生長)。
- **其餘一切是發想**,不是死規則:吃中線、預告、流血鐘、連段、三對三輪換、場地資源……每條的價值與坑收在 [[0_發想索引]],舊 v1 稿全文歸檔 `9_系統/_歷史/`。
- **骨架與發想之間隔著七道拍板題**([[0_拍板順序總表]]):每個發想都在等某道題的答案,每道題拍板後結論寫回確立項的分域檔。這就是設計推進的流水線——發想 → 過題 → 升格為確立項。

心智圖把這個結構畫成一張,四支與資料夾一一對應:

![[DIGO_EXE_設計心智圖.png]]

圓心是三支柱(憲法);左支是已確立的骨架;右支是七道拍板題,照「上游決定下游」排序;下支是排隊等拍板的發想;上支是拍板前要過的養分與約束。

## 從哪裡讀起

- **第一次進來**:本頁 → [[0_核心確立項]](現在確定了什麼)→ [[0_拍板順序總表]](接下來要決定什麼)。
- **要拍某道題**:先過 [[1_設計約束與審查備查]] 的憲法兩條與相關存活條,再進題頁看候選方案。
- **想找某個舊機制**(吃中線、流血鐘、三角色…):[[0_發想索引]],每條都標了它在等哪道題、有哪些已知的坑。
- **想看設計哲學的根**:[[8_設計反思]] 原文(迴避與走位、二選一真假、連段、輪轉替除四問),它在新框架下的形態見 [[1_設計約束與審查備查]] 第二節。
