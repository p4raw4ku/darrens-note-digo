# 設計約束與審查備查

> 這一頁是**拍板前的檢查站**。任何一道拍板題要動之前,先來這裡走一遍:第一節的**憲法兩條**每題必過;第二節是 [[2_設計反思]] 四點在新框架下的形態(原文一字未動、是養分,這裡是它的「現在式」);第三節的**存活條**在你動到對應領域時逐條核對(它們是機制數學,不會過期);第四節的**備查條**平時沉睡,發想回鍋或拍到對應題時被喚醒。審查原始全文與校核意見存於對話紀錄與本地存檔,結論以本頁為準。
>
> 使用順序:拍題 → 過憲法 → 查該題掛的存活條 → 看有沒有備查條被喚醒 → 進 [[0_拍板順序總表]] 的題頁。

## 一、憲法兩條

1. **不同層的東西,別擺在同一條軸上比。** 迴避對走位、移動面對效果面、連段對單卡、輪轉對拖長——歷來的疑難,拆層之後都解了一半。新架構照此分層:行動經濟層、晶片層、領航員層、場地資源層,各自獨立拍板、不互相混層。
2. **真正的引擎是底層的盤面壓力與資源稀缺,招牌機制只是顯示器。** 新框架下資源稀缺的實體=**手牌經濟**(手牌=行動預算/移動燃料/彈藥/切入子彈,2026-07-11 隨新回合結構更新;行動槽已廢止)+場地資源(題6 未定);盤面壓力的實體=幾何條款(題5)。拍任何表層機制前,先確認底層兩個支柱撐不撐得住它。

## 二、設計反思四點的新框架再讀

### 反思一(迴避對走位)→ 三層防禦的分工題

防禦供給變三層:數值防禦(閃避/護盾/力場,被動不佔行動)、走位(主動、佔行動)、換角(若題2 採三對三)。一回合能移動加攻擊後,走位閃避變弱、數值防禦份量上升。反思一的核心邏輯「功能重疊=假選擇、要用不可比區分」要在三層之間重演一次;特別警戒換角若免費會支配走位。「確定性優於擲骰」的論證原封沿用(見存活條 2)。

### 反思二(二選一是否真取捨)→ 取捨的外包對象多了行動經濟

舊結論「取捨外包給盤面壓力」仍對,新框架再加一句:**行動槽若不稀缺,移動面根本不會被讀到**。舊批判說衝鋒型全同步等於承認二選一是負擔;新框架下若一回合本就能移動加用晶片,全同步不再是破例——同一枚硬幣,都在說行動經濟越寬、二選一越沒地位。口訣化警戒與三層驗收尺照用(存活條 5),等題1 拍板後重量。

### 反思三(連段第三用途)→ 分層原則升格為架構原則

「效果空間扁」有了新出口:領航員能力與資源系統,不必硬塞第三用途。「單卡當下對跨回合序列不同層」正是本次重構的分頁邏輯。連段本身進 [[發想_連段與代碼]];若採三對三,[[發想_換角連攜]] 是更自然的連段表達。

### 反思四(輪轉替除防拖)→ 前提沒了、方法論全留

題目的前提(牌庫會耗盡)退回發想區,但方法論是四點裡最耐用的:重新定義成「怎麼保證遊戲會結束」、先跑分佈查詢再上機制、誘因對齊別罰錯人、防反向利用。用這把尺選新框架的反龜方案:互動產資源(誘因型)優於懲罰型時鐘;同時帶上它的警戒——「餵資源」是資源制的反向利用,設計時先想。

## 三、審查存活條(帶著走的六條+一條教訓)

這批綁在確立項(傷害乘以次數、三防禦機制、晶片二選一、猜想為核心樂趣)上,多為機制數學,不隨規則稿改版失效:

1. **多發攻擊要有載體**:相剋要成立,「攻擊次數大於一」的形狀必須真的存在於卡池,否則閃避沒有天敵(對閃避 6,任何單發約九成機率整張蒸發)。三條數值帶要協同:傷害帶地板與力場帶天花板不可同值。→ 用在題4、發想_舊範例卡表。
2. **確定性閃避要換框架**:「抵銷前幾擊」的三種具體化各自退化——加必中則反轉「閃避剋大擊」、按攻擊重置則成免疫牆、跨回合點數池則被便宜薄擊墊刀。候選框架:**超額削減**(超過閃避值的部分砍半)——零亂數、剋大擊非免疫、不可墊刀。→ 題4。(2026-07-10 更新:閃避已拍**機率制 1/3＋額度消耗**為正典,超額削減降為替代框架備查。)
3. **力場與護盾帶值不可一比二對映**:護盾平均每擊只擋盾值一半(破盾與穿透輪替)、力場每擊全額擋,一半數值的力場就全面不劣於護盾,且護盾還可被「先削後重」排序剝削。候選解:力場過載窗(被兩倍重擊命中後下回合失效)。→ 題4。(2026-07-10 更新:過載窗經 user 否決——力場定純門檻、天花板暫定 5 且多數領航員為 0;同值死角改為刻意設計＋池硬約束,見 [[確立項_攻擊式]]。)
4. **防禦常駐明牌=整場只讀一次**:相剋只在配對層轉、對局內凍結。解法:三對三換角(題2)讓防禦輪轉,或讓防禦有可見狀態變化。→ 題2、題4。(2026-07-11 更新:3對3 已拍板、防禦輪轉成立;閃避額度的回合內消耗又添一層可見變化——本條大半已解,殘餘看題4 帶值驗收「防禦參與猜想」。)
5. **驗收用三層尺,單一比例量不出口訣化**:①兩面期望值貼近率(真兩難密度)②只用可見盤面特徵預測最佳解的準確率(九成以上=口訣化,不放行)③換一組對手手牌後最佳解翻轉的比例(預判真的參與決策的證據)。→ 題1 之後的所有驗收。
6. **勝率驗收要有檢定力**:五十局的誤差約正負十四個百分點,判定窄目標帶要每配對約兩千局(模擬);並列「賣點出場指標」——勝率及格但特色沒出場=假豐富。真人數十局改測手感。→ 領航員與構築驗收。
7. **距離錨點教訓**:範圍規則的第一句就寫死「從角色所在格起算」——同一條文的兩種錨點讀法是兩款不同的遊戲。→ 題5。

## 四、審查備查條(十三條:一句話+喚醒條件)

| 備查 | 一句話 | 喚醒條件 |
|---|---|---|
| 預告時序 | 字面讀法=保證命中、放寬讀法=必閃,兩頭站不住;修正=第二回合到期+形狀覆蓋 | [[發想_預告攻擊]] 回鍋 |
| 吃中線安全屋 | 領地變深、射程鎖死 → 打不到的聖域;替代=視窗滑移 | [[發想_吃中線]] 回鍋 |
| 冷卻計時主體 | 「間隔一回合」不寫誰的回合,絕對回合讀法讓鎖形同虛設 | 任何帶時效的規則 |
| 流血鐘偏袒領先方 | 對稱扣血下血量差不變,領先方龜到贏;修正=造傷豁免 | [[發想_流血鐘]] 回鍋 |
| 雙空殘局座位奇偶 | 強制出牌+固定補牌下,誰先斷牌開局就注定、先手輸在座位 | 題3 選耗盡型供給 |
| 回收補貼龜縮方 | 移動牌無限回收=發武器給血量領先的龜縮方 | 題3 選混合型供給 |
| 先手補償零粒度 | 「禁一整回合傷害」是開關不是旋鈕,且本質是移交第一刀權;改連續量(首回合傷害上限) | 題1 拍板後設計先手補償 |
| 破例強度耦合 | 破例的價值隨核心硬度水漲船高,核心未定時標不了價 | 設計領航員能力 |
| 反制成本 | 「受任何傷害即中斷/重置」=零成本關掉對手特色;打斷要有門檻或代價 | 設計蓄力、朝向類能力 |
| 內容預算 | 同名上限+牌庫張數決定卡池最低名數;內容跟不上,構築自由是幻覺 | 設計構築層 |
| 代碼死欄 | 全卡池付印、沒有規則讀取的欄位是純認知負擔 | ✅ 已解除(2026-07-10 連鎖與切入讀取代碼) |
| 護欄要可機驗 | 構築限制的類別必須用卡面欄位定義,「主用途」判不了 | 設計構築護欄 |
| 結算時序單一模型 | 傷害的結算時點(當場/延後)全書只能有一個模型;每筆傷害掛全局序號 | 寫引擎狀態機 |
