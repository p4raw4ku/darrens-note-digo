# B6 尺 v2（metrics 擴充）報告

工作目錄：`round1/work/b6_engine_v2/`（`cp -R engine_base` 而來）。只改 `core.py`（最小掛點）＋`metrics.py`（新函式），新增 `metrics_v2_tests.py`。單進程、無 multiprocessing、無子 agent，全程只讀 vault、只寫 `round1/` 底下檔案。

## 一、最小掛點說明（為何必須動 core.py）

引擎原本沒有逐回合 HP 軌跡——`GameRecord` 只留 `final_hp`（終局快照）。四個新指標全部需要「adv(t) 隨回合變化」，因此在 `core.py` 加了三處純加法、不碰任何結算邏輯的掛點：

1. `GameRecord.__slots__` 加一個新欄位 `hp_by_round`：`[(round_no, {0: 隊伍HP總和, 1: 隊伍HP總和}), …]`。
2. 新方法 `Engine.team_hp(st, side)`：讀 `st.units[side].hp` ＋存活後備 `st.benches[side]` 的 hp 加總（方向二三 benches 恆空，等同單一 HP；方向一三名合計）。純讀取，不改任何欄位。
3. `Engine.play()` 內三處 `append`：開局後一次（round 0，開局快照）、每回合雙方走完＋安全網結算後一次、收尾補一次（防止「第一手就 KO」導致序列缺尾）。

`diff` 只動了 `GameRecord.__init__`／新增兩個方法／`play()` 內三行 append，未改動 `apply_safety_net`、`_do_attack`、`resolve_attack` 等任何傷害／勝負判定邏輯。因為 `V11Dir3Engine`／`V11Dir2Engine`／`V11Dir1Engine`（`v11_calibrate.py`）與 `Dir1Engine`／`Dir2Engine`／`Dir3Engine`（`setups.py`/`dir3.py`）都沒有覆寫 `play()`，此掛點對三方向、v1/v1.1 各檔位全部透明生效（已用 dir1/dir3 各跑一局手動核對，見下）。

`metrics.py` 只在檔尾新增一段（`_COMBAT_DAMAGE_EVENTS` 起），舊有全部函式（`game_length_stats`／`first_mover_winrate`／`movement_metrics`／`chase_deadlock_rate`／`dilemma_density`／`playbook_score`／`hand_sensitivity`／`matchup_health`／`change_navi_stats`／`resource_pickup_stats`）一字未動。

## 二、四個新指標——定義最終版

### 1. 優勢軌跡 `advantage_series(rec)`

```
adv(t) = (甲方隊伍HP總和(t) − 乙方隊伍HP總和(t)) ÷ (開局雙方隊伍HP總和 ÷ 2)
```

- 分母固定用 **round 0（開局）快照**，不隨戰局變化——三方向通用（方向一三名合計進分母，自動放大到與隊伍規模成比例）。
- 回傳每局 `[(round_no, adv), …]`，t=0 為開局（恆為 0）。
- 是尺②③翻盤率／張力三量的共同地基，其餘兩指標都是這條序列的衍生統計。

### 2. 翻盤率 `comeback_rate(records, threshold=-0.20)`

判準：勝者在 **t≥2** 的任一回合，其**勝者視角** adv（勝者=甲用原值、勝者=乙取負號）曾 ≤ −0.20。平局不計入分母。

- `t≥2` 是刻意的：t=0/1 的「深度落後」多半只是開局隨機性或先手單次施法的雜訊，不算真正意義下的「一度大幅落後又逆轉」。
- 分母＝有勝負的對局（`winner is not None`）。

### 3. 張力三量 `tension_metrics(records, zero_band=0.02)`

- **(a) 波動度**：`|adv(t)−adv(t−1)|` 對全部對局、全部相鄰回合對取全局平均（不是先局內平均再跨局平均——用全部 delta 攤平，樣本數＝Σ(每局回合數−1)）。
- **(b) 領先易手次數**：把 adv 序列的符號（`|adv|<0.02` 視為零帶、忽略不計號）取「最近一個非零帶符號」比較，符號翻轉才算一次易手；首次脫離零帶不算易手（沒有前一個非零號可比）。避免零帶抖動灌水。
- **(c) 決勝時點**：最後一次符號跨越（用同一套零帶邏輯）發生的回合 ÷ 對局總回合數。無任何跨零的對局（單邊碾壓局、或全程在零帶內）不計入這條分母，另外回報「無跨零局數」與「決勝時點涵蓋局數」讓讀者看到涵蓋率、不會誤讀成全樣本統計。

### 4. 安全網決定勝負率 `safety_net_decisive_rate(records, loser_share_threshold=0.5)`

分子＝對局同時滿足下列任一：
- **(a) 致勝一擊來源＝安全網**：敗者在「安全網」事件發生的那個回合、回合結束時 HP 已到 0（用 `hp_by_round` 直接讀，不重算結算細節）。
- **(b) 敗者總失血中安全網佔比 ≥ 50%**：用 `_safety_net_loss_by_side()` 從 `hp_by_round` 的逐回合 HP 差扣除「當回合對手實際造成的戰鬥傷害」（讀 `自攻`／`晶片能力`／`領航員能力攻擊` 三類 log 事件，三方向共用 schema）反推安全網淨損失，對每個「安全網」事件回合分別累加兩側。

分母＝有勝負的對局。**同時並列輸出舊「安全網觸發率」**（`game_length_stats` 既有欄位，改用判定局數當分母以利對照）：觸發率只問「安全網機制這局有沒有介入」（門檻寬鬆），決定率問「安全網是否真的主導了誰輸誰贏」（門檻嚴格）——兩者故意並陳，避免把「觸發」誤讀成「決定」。

**已知限制（見下方可信度聲明）**：`_safety_net_loss_by_side` 的扣除法假設「戰鬥傷害與安全網損失在同一回合不會被同一次 HP 變動糊在一起而無法拆分」——這在核心引擎的豁免規則下（造傷者本回合免安全網）理論上不可能同源疊加到同一格，已用 300 局 demo＋dir1/dir3 手動抽驗一致，但沒有對抗驗證過陷阱傷害（`陷阱觸發`，log schema 不同、無 `side`/`round`/`damage` 三件套）與安全網同回合並存的邊界情形——dir3/dir1 v1.1 демо 未出現此組合，留給主幹對抗校核。

## 六、可信度聲明

- **樣本**：dir3 v1.1 demo n=300（seed 20266711 起連續遞增，先手輪替），單一配對（greedy 對 turtle）、單一防禦檔位（護盾/護盾）。翻盤率／張力三量／安全網決定率的分母（有勝負局數）為 242，非 300（58 局平局需留意，尤其張力三量的「決勝時點」分母又進一步收斂到 164 局涵蓋率 68%）。
- **CI**：本報告未計算信賴區間（規格未要求；n=300 對翻盤率 16.1% 之類的比例估計，粗略二項式標準誤約 ±2.3 個百分點，僅供參考，非正式 CI）。
- **已知偏差**：
  1. `安全網決定勝負率`的「路徑(b)佔比」計算依賴戰鬥傷害事件三件套（`自攻`／`晶片能力`／`領航員能力攻擊`）皆有 `side`+`damage`+`round`；陷阱類事件（`陷阱觸發`，方向二專屬）目前不在扣除範圍內，若某局安全網回合恰好同時有陷阱傷害，佔比計算可能失真（本 demo 為方向三、無陷阱，未觸及此邊界；方向二／A6 實驗使用此引擎時務必留意，建議主幹或 A6 worker 在跑方向二資料時另做抽驗）。
  2. `張力三量`的「決勝時點」只對「有跨零」的對局計算，本 demo 涵蓋率 68%（164/242）——結論方向性成立，但涵蓋率非 100%，不宜引用「決勝時點=0.304」當全樣本敘述，需帶著「僅涵蓋跨零局」的限定詞。
  3. 「安全網觸發率＝安全網決定勝負率」在本 demo 與另一組 greedy 鏡像抽驗（n=200）中都恰好完全相同——是否為此檔位（v1.1、dir3）的普遍現象、抑或抽樣巧合，未跨方向／跨檔位驗證，屬素材、待主幹在其他實驗（尤其 A5／A6 用到 B6 metrics 時）交叉確認。
  4. `hp_by_round` 快照時機是「雙方都走完＋安全網結算後」，不含回合中途（一方走完、另一方尚未走）的瞬時 HP——`advantage_series` 的 t 軸粒度是「回合」而非「半回合」，這是規格原文「逐局輸出 adv 序列」的自然讀法（以回合為單位），但若後續實驗需要半回合粒度需另外加掛點（目前未加，屬本次範圍外）。
- **方向性結論**：本報告只提供 dir3 v1.1 單一配對的 demo 數值，供主幹核對指標實作正確性與初步量感；**不對「三方向普遍模式」下任何結論**——那是 A5／A6 用這批 metrics v2 出貨後才能講的話。

## 七、快速使用範例

```python
from core import GameConfig
from setups import build
from policies import make_policy
import metrics as M

cfg = GameConfig(direction=3, hp=100, ...)   # 或用 v11_calibrate 的 v1.1 config
eng, setup = build(3, cfg, def_a='護盾', def_b='護盾')
rec = eng.play(cfg, setup, make_policy('greedy_damage'), make_policy('turtle'))

M.advantage_series(rec)                 # [(round, adv), …]
M.comeback_rate([rec, ...])             # {'翻盤率':…, '判定局數':…, '翻盤局數':…}
M.tension_metrics([rec, ...])           # 波動度／領先易手次數／決勝時點
M.safety_net_decisive_rate([rec, ...])  # 安全網決定勝負率＋舊觸發率並列
```
