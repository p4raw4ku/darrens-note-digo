# 三方向驗證導讀：七題用三方案答，答案是什麼，最後回扣三方案

> 2026-07-05 一整夜三輪驗證的因果紀錄。故事的形狀：**七道拍板題還沒答案 → 用三個候選方案各給一整套答案 → 大規模實測看哪套站得住 → 每道題各自撈到什麼 → 回頭判三個方案的生死**。本文照這條線走，行話都在第三節先翻成白話；粗資料（規格、卷宗、搜索報告原文）在 `_三方向驗證存檔/`，第九節有逐檔地圖。只要最終判定，讀 [[5_三方向驗證結論]] 即可。

## 一、起點：七道題的上游還有一道題

2026-07-03 框架重整後，設計只剩骨架（確立項）＋七道拍板題。但七道題的上游還有更大的未定：**核心樂趣三支柱誰當主角**——同一副骨架，主角不同會長成三款不同的遊戲。討論收斂成三個成套方案：

- **方向一「讀心換角戰」**：猜想領軍。3對3 輪流上陣，賭「換角預測」是讀心樂趣的主引擎（寶可夢式）。
- **方向二「走位晶片棋」**：盤面壓力領軍。1對1＋場地資源＋隱形陷阱，賭「幾何＋資源」能把移動撐成真選擇。
- **方向三「極簡核心」**：選擇純度領軍。零附加系統的最小一致核心，賭骨架自己就能及格——同時兼任另兩案的對照組。

user 拍板：除確立項外，所有未定項由 Claude 發想填空、補成三套**可模擬的完整規格**、實測定生死。

## 三、看懂數據前，先把行話翻成白話

粗資料裡大量出現以下詞，正文第一次遇到也會就地簡註。

**電腦對手（五支基線策略）**——所有勝率都是讓這些「固定打法的電腦」互打出來的：

| 名稱 | 白話 |
|---|---|
| random_legal | 亂走，但每步合法。當「地板」用：任何正經策略都該贏它 |
| greedy_damage（貪傷） | 每回合挑「當下期望傷害最大」的行動。代表無腦進攻 |
| turtle（龜縮） | 拉開距離、只打零風險的目標。代表消極防守——它贏太多＝遊戲會悶 |
| positional（搶位） | 最大化射程覆蓋與機動。代表走位流 |
| oneply（一步前瞻） | 枚舉自己整回合所有行動組合、並模擬對手最佳回應後選期望最好的。最聰明也最慢（單局可達數十秒），所以很多實驗只能少跑它 |

**統計詞**：

| 詞 | 白話 |
|---|---|
| n=1000 | 跑一千局。局數越多數字越可信 |
| CI95±3pp | 95% 信賴區間、誤差上下 3 個百分點。「±15pp」代表誤差大到 0.49 跟 0.70 可能其實分不出來 |
| 平局折半 | 勝率＝勝÷(勝＋敗＋平×0.5)，平局各算半勝 |
| side-swap 公平化 | 同一組對手先後手各打一半，避免把先手優勢誤算成策略強度 |
| 加權平均 | 各配對的數字按局數加權後平均 |

**機制詞**：

| 詞 | 白話 |
|---|---|
| 安全網 | 對局拖太久時，系統從第 N 回合起每回合扣雙方血逼出勝負的保險絲（該回合有造成傷害者豁免）。觸發率高＝遊戲常拖到要保險絲介入 |
| 漏洞回歸 | 把已知的賴皮打法寫成程式，在新數值下重新下場打——還能穩贏就是洞沒補上；勝率壓到 0.55 以下才算「關閉」 |
| 追逐死結 | 連續多回合雙方都造成零傷害的空轉（你追我跑、誰都開不了火） |
| 回復牆／閃避牆 | 兩個實錘賴皮打法：疊回復龜到天荒地老；靠高閃避站著不怕打 |
| turtle 對 greedy | 純龜縮對純進攻的勝率。健康帶 43–57%：龜贏太多＝獎勵不互動；龜輸太多＝防守無意義 |

**量測與方法詞**：

| 詞 | 白話 |
|---|---|
| 尺①真兩難密度 | 出晶片的時刻裡，「兩面都想用」（兩面估值差兩成內）的比例。越高＝招牌二選一越是真選擇 |
| 尺②口訣化 | 只看公開盤面就能猜中行動的程度。≥90%＝玩家在查表不是在思考，不放行 |
| 尺③手牌敏感度 | 換一手牌會不會改變最佳行動的比例。越高＝隱藏資訊（手牌）越承重、越值得猜 |
| 座標下降 | 調參法：一次只動一個參數、變好就留下，輪流動每個參數直到動什麼都不再變好 |
| 目標函數 | 把「好遊戲」定義成一個可計算的分數（長度貼目標＋龜縮不佔優＋先手公平＋安全網少介入），越低越好 |
| 熵（bits） | 選擇分散程度。越高＝攻擊形狀選用越平均、打法越不單調 |

**共同目標帶**（每個實驗都對著它們驗收）：對局長度中位 8–14 回合｜先手勝率 45–55%｜turtle 對 greedy 43–57%｜安全網觸發率 <30%（>50% 重罰）｜漏洞回歸 <55%。定義原文見 [[engine_contract]]。

### 4.4 第一輪收斂：病根是校準、不是方向

三方向**全面偏防守**：中位 33–46 回合（目標 8–14）、安全網幾乎每局介入、龜縮＋回復壓過所有進攻。判定為**數值校準問題**（攻擊穿透量遠低於 HP＋防禦＋回復的總和）而非方向問題——這個判斷直接生出第二輪的平衡搜索。工程教訓：三個方向的實驗同時搶 8 核（負載一度 91–172）拖垮方向二→第二輪起全員機器紀律（[[stage2_簡報]]）。

## 五、第二輪：量出階梯、搜出 v1

### 5.1 三把尺工具與「真兩難密度階梯」

### 5.3 v1 平衡搜索（[[搜索_v1報告]]）

- **要驗證什麼**：v0 三大病（過長／安全網接管／龜縮壓制）能不能**只調數值**治好。
- **變因（10 維）**：HP、閃避帶、護盾帶、力場帶、閃避結算模式（機率制／現行確定性／字面修正版）、領航員自攻、迴復裝置檔位、安全網起點、安全網模式、先手補償。
- **方法**：方向三當校準場（最便宜最乾淨）、座標下降、每評估 600 局、共 41 次評估。
- **軌跡**（分數越低越好）：起點 2.186 → 護盾帶降 10 → 自攻升 15×1 → HP 降 100 → 迴復改每局限 2 次 → 安全網起點提前到 15 → 補償改「首回合 1 行動」→ 自攻退回 10×1（HP100 下 15×1 把先手推到 0.599——座標下降抓到的維度交互）→ 終點 **0.484，分數降 78%；對局中位 21.5→11.5**。
- **翻案（本輪最重要的一課）**：便宜策略把前 5 名判成同分（互打雙方都沒帶閃避、閃避模式根本沒上場），但**漏洞回歸把三種閃避結算分得一清二楚**：

| 閃避結算語義 | 閃避牆回歸 | 回復牆回歸 | 判定 |
|---|---|---|---|
| 字面修正版（小擊全擋＋超額砍半） | 0.924–0.993 | 0.362 | 閃避變無敵牆 |
| 機率制（確立項現值） | 0.780 | 0.502 | 牆關不掉 |
| **現行確定性（實傷＝閃避值＋超額一半、永不歸零）** | **0.172** | **0.012** | **唯一同時關掉兩個賴皮** |

**方法論教訓：對抗攻擊抓到的問題是真的（兩讀確實是兩款遊戲），但攻擊者提的修法未必是答案——選邊要靠漏洞回歸實測。** 冠軍配置：閃4盾10場5／HP100／自攻10×1／迴復回30限2／網起15遞增／補償一行動——中位 8.0、先手 0.535（n=1000）、turtle 0.612、安全網 0.413。

### 5.4 移植驗證（[[移植驗證_v1]]）：校準不能跨方向照搬

冠軍配置搬到方向一（HP 等比縮到每名 47）與方向二（HP 原封）——**沒有一個方向全數達標**：方向三自己 2 項未達（turtle 0.612 偏龜、安全網 41%）；方向一 3 項（長 16、**turtle 反轉到 0.331**＝龜縮反而大輸、安全網 54%）；方向二 2 項（長 18、安全網 83%）。**病根診斷**：安全網沒隨尺度重校——方向一 47 血的角色對「基值 10 每回合遞增」的安全網，連續三次觸發就團滅一名；方向二血量沒變卻 83% 觸網＝節奏**天生慢**（資源／陷阱／位移的決策點多）。報告開出逐方向修正建議→第三輪任務書。

## 七、答案：七道題各自撈到什麼（詳文見各題頁末「模擬驗證素材」段）

- **[[題1_行動經濟]]**：甲案（2 行動、同類至多 1）全程實測可行、追逐死結沒成為問題。先手補償最意外——「首回合 1 行動」在方向三最佳、在方向一反而最糟（0.629）→**補償檔位要跟方向一起拍，不能全域定死**。行動經濟自己撐不起二選一（尺① 0.019）。
- **[[題2_隊伍規模]]**：3對3 的主賣點（換角讀心）機器測不出承重（0.495）；範圍攻擊在一次一人上場下全數退化單體；雪球 0.318 存在非致命。換角＋能量把真兩難推到全場最高（0.837）→**素材珍貴、當主軸風險大**。
- **[[題3_晶片供給]]**：循環型供給全程跑通、斷牌問題如預期迴避；「迴復每局限次」是平衡搜索第三大槓桿；尺③（0.78–0.98）證明手牌是真的猜測材料。
- **[[題4_防禦數值]]**：收穫最大的一題——閃避結算三讀法實測翻案（唯一雙關賴皮的是現行確定性讀法）；護盾必須寫死「盾量跨回合保留」；v1.1 帶值閃4盾10場5；「攻擊對防禦」與「防禦對防禦」兩層都要驗（指標盲區）；力場過載窗高頻可行。
- **[[題5_幾何條款]]**：覆蓋矩陣證明站位有真差異、無死角；鎖列要改動態（靜態最前列會系統性打空）；近斬對後兩列絕對打不到——近戰卡的定位是題5×題6 交叉課題。
- **[[題6_資源經濟]]**：誘因型反龜有實據（承重 +0.526、罰龜 −11.2pp）；但目前是「勝負開關」不是「節奏紅利」（懂對不懂 0.99）——**紅利化是本題拍板前最大功課**。
- **[[題7_對局長度]]**：「拖長是數字問題」獲直接證實（不動機制、五根數值槓桿把 33–46 壓到 8–11）；安全網被回復牆反用過（防反向利用的活教材）；「觸網率」這個指標本身該重定義；長度目標帶可能要逐方向訂（方向二天生慢）。
