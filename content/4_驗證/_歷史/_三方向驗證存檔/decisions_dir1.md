# 方向一 讀心換角戰——阻斷／重要問題處理決策記錄

> 依規則律師 `rulelawyer_方向一_findings.md` 逐項處理。原則：選最簡單且不違反存活條的讀法，記錄理由；不自行拍板遊戲平衡數值（那是判斷/品味類決策，留給 user）。

## 阻斷級

### A1 換角當回合的行動計數（換角＋新領航員能力/攻擊是否合法）

**決策：採讀法一（換角吃掉整個「領航員」類，換角當回合不能再用能力或攻擊，只能再配移動或晶片）。**

驗證：實測 `engine/setups.py` 的 `Dir1Engine`／`core.py` 共用機制——`Action.category()` 對 `kind='領航員'` 的所有 subkind（攻擊/能力/換角）一律回傳 `'領航員'`，`legal_action_pairs` 的 `used_categories` 排除同類。故換角（`subkind='換角'`）與同回合再選任一 `kind='領航員'` 的行動，因同屬 `'領航員'` 類而被自動排除。實測 `legal_action_pairs` 中「換角接領航員類第二行動」的組合數＝0（見下方驗證片段）。

```
單次 legal_actions 領航員子類分布: {'換角': 2}   # 換角後，同回合第二格 legal_actions 只剩 pass/移動/晶片
換角+領航員第二行動組合數: 0
```

此為共用核心（`core.py`，不可修改）既有機制自動落實的結果，且與規則律師建議的最保守讀法一致，不需要在 `dir1.py` 額外程式碼介入。**已在方向一 overlay 及回歸測試中確認生效。**

### A2/A3 變體B超額削減「有效」指涉、「全額」指涉不明（共用基準 §四、core.py 傷害結算）

**決策：保留 `core.py` 現有實作，不繞過、不修改，記錄現有語意作為方向一採用的權威讀法。**

`core.py::resolve_attack` 的超額削減分支現況：
```python
if per_hit_evasion > 0 and d > per_hit_evasion:
    over = d - per_hit_evasion
    d = per_hit_evasion + (over + 1) // 2   # 有效＝閃避值 + ⌈over/2⌉，此值就是「實傷」
# d ≤ 閃避值時：d 不變（全額命中，非全額抵銷）
```

即：
- A2（「有效」指涉）：現有實作把公式值**直接當實傷**（`d = 閃避值 + ⌈(單傷−閃避值)/2⌉`），等同規則律師建議修法 A2 的效果（實傷式）。**採用此讀法。**
- A3（「全額」指涉）：現有實作是「單傷 ≤ 閃避值 → 全額**命中**」（`d` 不變、不清零），**不是**規則律師建議的「全額抵銷（實傷0）」。

**為何不依建議修法而保留現況**：這是 `core.py` 的傷害結算核心，契約 §一明文「移植 `sim_combat.py` 的 `resolve_attack` 逐擊狀態機⋯語義不可改」，且任務指示「共用檔 core.py⋯一律不得修改」。回歸測試 `超額削減 5×3 對閃避8 → 15`（`tests.py` §6.1）已鎖定「全額命中」讀法為權威版本並全綠通過，不屬於方向一可自行決定或繞過的範圍。

**影響記錄（供漏洞獵殺留意）**：在此讀法下，變體B「超額削減」對小擊（≤閃避值）不設防禦（全額命中），只對大擊砍半。此讀法削弱「多發載體剋大擊」的存活條1效果——與規則律師 A3 標記的疑慮一致：小擊仍會全額累加傷害、閃避在變體B下對機關槍（5×5）、三連彈（10×3）等多發低單傷武器**幾乎無防禦力**。此為已知現有核心行為，非方向一模組的 bug，已記錄不予繞過。

## 重要級

### B1 HP帶：共用基準150 vs overlay每名70×3＝210，長度目標未重算

**決策：方向一總血210（每名70×3，覆蓋共用基準150），對局長度目標與安全網起點不套用共用基準「中位8–14回合／第25回合」的舊校準值；改以本次實際模擬跑出的中位數與安全網觸發率如實回報（見 `摘要_dir1.md`）。**

不修改 `safety_net_turn` 預設值（=25，來自 `GameConfig` 預設），因為共用基準本身把第25回合當「實驗參數」而非鐵律；只在指標詮釋時不生搬硬套舊目標帶。

### B2 領航員自身攻擊固定10×1對力場帶存在完全免疫盲區

**決策：不修改 `DIR1_NAVIS` 數值（此為遊戲平衡判斷/品味類決策，不由本次驗證任務自行拍板），如實記錄此盲區為異常項於摘要與 anomalies，留給規則設計者／user 決定是否調整（例如自身攻擊提高到 16×1 以上以穿透力場天花板15，或改變攻擊次數）。**

實測：自攻10×1對力場10/12/15 三檔實傷皆為0（`resolve_attack` 逐擊固定減力場、力場≥單傷則整擊歸零）。兩個砲型（力場12）互打自身攻擊＝永久零傷僵局，只能靠晶片/能力破局。摘要中將以「自攻對力場穿透率」指標呈現實測數字。

### B3 範圍攻擊在「一次僅1名上場」下退化為單體，相剋矩陣測不出範圍價值

**決策：方向一專屬相剋矩陣沿用共用 `ATTACK_SHAPES`／`DEFENSE_CONFIGS`（隔離結算，非對局內），本身就是單體結算、不受「一次1上場」影響，數字仍有效；但在摘要中明文標註：方向一**對局內**（`exp1`/`exp2`）的範圍攻擊（十字/鎖排/鎖列/貫穿/近斬寬）因場上恆1個敵方單位而退化為單體，不得用對局指標宣稱範圍形狀已被驗證出差異化價值。相剋矩陣（隔離結算）本身測的是無關「上場人數」的傷害相剋，不受此限，仍具參考力。**

### B4 力場過載窗觸發時序（觸發擊本身是否仍過力場）

**決策：採用 `core.py` 現有逐擊時序——觸發擊本身仍先固定減力場值（`pre_field_d`，過力場前的單次傷害，用來判定是否 ≥ 2×力場值），該擊仍照常扣力場；力場失效旗標在該擊結算「之後」生效，故同一次攻擊後續的擊才會直接全額命中。回歸測試 `力場過載窗 25×2 對力場10 → 40`（首擊 25−10=15 進血並觸發過載；次擊力場已失效，25 全額）驗證此時序，全綠通過。不繞過、直接採用。**

## 次要級（記錄不逐一開新決策段落，摘要中一併註記）

- C1 盾能重整「若力場失效則恢復」：採 `core.py::_apply_chip_ability` 的 `shield_reset` 實作——立即清除 `field_disabled` 旗標（恢復力場減傷能力），採此為權威讀法。
- C2 換角「換入者力場過載解除」語意：確認現有 `hook_apply_change_navi` 對換入者的 `field_disabled` 重置為 False——雖然規則律師指出換入者本就非過載態（此句對換入者為空操作），但程式碼寫法對「換出者」的過載狀態不受影響（換出者被放回後備，過載計時器實質上隨之凍結，直到其再度換上場時才會依殘留旗標繼續）。此為現有行為，不修改。
- C3 「換入者護盾回滿」對無護盾（`shield_max=0`）領航員為空操作：確認 `shield_cur = shield_max` 對 `shield_max=0` 情形（斬型、砲型）僅是 `0=0`，無副作用、不報錯。
- C4 能量「唯一用途」與晶片「能力面」用詞：本次撰寫摘要與程式註解時，明確區分「領航員能力」（耗能量）與「晶片能力面」（不耗能量、耗牌），避免混淆。
- C5 換角能量變體（免費但費2能量）下的次數限制：**決策——採「免費換角仍受『領航員類每回合至多1次』約束」的保守讀法**（與A1一致的精神：換角變體只改「是否佔用行動格」，不改「類別限次」規則）。在 `dir1.py` 的能量制換角實作中，換角動作仍標記為 `kind='領航員'`（僅不消耗行動格，改用能量），因此同回合仍不能疊加領航員類的其他行動或二次換角。
- C6 先手補償×換角/傷害上限20交互：傷害上限20 對「上場者」生效（`core.py::_do_attack` 的 `cap` 只限制單次 `resolve_attack` 的 `loss`，天然只作用在被打的上場單位），已寫死、不需額外處理。首回合僅1行動下換角佔用該行動，此為既有機制自然結果，摘要中記錄觀察到的頻率。
- C7 強制換入的溢傷/同次攻擊時序：**實測驗證**（`_do_attack` 對 `10×3` 攻擊，受害者 5HP 清空防禦）——`resolve_attack` 對同一個 `defender` 物件一次跑完整 3 擊（30 傷害全記在原上場者身上，內部 `defender.hp -= loss` 使其變 0 或負值），`_check_unit_down` 在整次攻擊結算「完成後」才觸發強制換入。實測結果：原上場者（斬型）HP 歸 0、`alive=False`；新換入者（砲型）HP 維持滿血 70/70、完全未受該次攻擊任何一擊波及。**確認：同次多發攻擊的溢傷不會轉移到新換入的後備身上，換入者以滿血承接下一回合。** 此為現有行為，方向一 3v3 規格「HP各自計算」下合理，不改動。
