# 驗證索引

> 這個資料夾是**證據層**:拍板不靠手感、靠數據,模擬器跑出來的數字以這裡為準(工作準則見 CLAUDE.md「數值以模擬器為準」)。上游是確立項的攻擊式與防禦三機制(數據綁著它們,所以沿用),下游是拍板題(尤其題4 的三條硬約束全部源於這裡的數據)。

| 檔案 | 它是什麼(因) | 它支撐什麼(果) |
|---|---|---|
| [[4_實測紀錄]] | 模擬器逐輪驗證的條件/結果/結論——因為機制數學(護盾結算、相剋、奇偶)沒有乾淨公式,只能逐擊跑出來 | 題4 的三條硬約束(多發載體、閃避框架、力場護盾帶值)全部以此為據;輪2/點4 綁舊規則稿,結論已轉入備查 |
| [[5_傷害模型]] | 減傷三機制的名詞定義+單擊結算決策圖——結算規則的白話版與程式版之間的橋 | 未來規則引擎 `resolve_attack` 的骨架;拍題4 時的共同語言 |
| [[6_傷害線圖]] | 攻擊次數對期望傷害的線圖(單機制各檔位)——把結算數學畫成可比對的曲線 | 題4 定數值帶時的參考圖:每種防禦的懸崖與甜蜜點一眼可見 |

## 使用時機

- **拍題4 之前必讀**:三張線圖+實測結論摘要,配 [[1_設計約束與審查備查]] 的存活條一起看。
- **改任何結算規則之後**:重跑對應輪次,數據對不上就是回歸破壞(模擬器傷害核心=規則引擎第 0 版,要守確定性)。
- 盤面與行動層的驗證(走位、行動經濟)等題1、題5 拍板後隨模擬器重寫一起建。
