## 四、駕駛(模擬對局的出牌策略)

> 駕駛=量測儀器的一部分:換駕駛=換 meta,數字會動(先攻優勢隨 meta 翻轉=層1_2 實例)。**結論引用一律標駕駛。**

### 共通框架(三檔皆同)

- 對齊才攻擊(層1 直線最小版=同橫排命中);未對齊→挑移動力夠的牌用移動面對齊(優先燒非攻擊牌)。
- **切入/接手**:對方連鎖中,手上有接得上的攻擊牌就切,挑**總傷害最大**的(切入=機會傷害、不做鏈規劃)。
- 後手第一回合洗二:洗回與其餘手牌「連結數最少」的兩張。
- 不主動迴復(僅鏈向留牌在清手位時用回復卡;例外=回復優先駕駛,見該節);層1 無換角、無質變。

### 貪婪(Greedy)——對照駕駛

1. **開鏈**:挑手上**總傷害最大**的攻擊牌打出——完全不看接續。
2. **續打**:接得上的攻擊牌中再挑總傷害最大;沒有→收手。
3. **永不留牌**:只要手上有攻擊牌就出手,單張也照打。
- 行為畫像:鏈常一張(常開在梯尾、白白丟掉接續);殘手單張全數照丟;治癒類永遠堵在手上(全三組成下=堵死到回合上限,層1_3_1 實錄)。
- 用途:層1 一輪/二輪的基準、各測試的對照駕駛。

### 鏈向(ChainGreedy, hold=False)

1. **開鏈**:把手上字母攻擊牌按代碼排好,找**最長可打段**(同碼堆疊+逐碼爬升、米號牌全插),從該段**最低碼**開打。
2. **續打**:**照序**——同碼優先、再打次一碼(照排序打即可把整段打完);字母用盡才墊米號牌。
3. 仍不留牌:湊不出兩張也照打單張。
- 與貪婪的差異=開鏈「最大傷→最長串起點」、續打「最大傷→照序」。
- 效果(v1 組成):鏈均長 1.46→1.86、切入 5.1→9.3/場、連鎖佔比 24%→33%。

### 鏈向留牌(ChainGreedy, hold=True)——層1 量測標準

1–2 同鏈向。
3. **留牌**:手上最長可打段 <2 → 收手不打,把單張留到下回合湊對(=已拍「收手留牌保質」規則的用法)。
4. **防堵手**:手滿(上限 8)仍湊不出兩張 → 清一張騰位——受傷且有回復牌先打回復,否則打出最弱單攻。
- 與鏈向的差異(v1 組成):鏈均長 1.86→**3.13**、連鎖佔比 33%→**98%**、對局中位 5→11 回合、切入 9.3→14.7/場;先攻 50.5%→46.1%(後攻微利,層1.3 複驗)。
- 定位:對齊兩個既有前提(打滿假設+收手留牌規則)的**量測標準**——不是「最強 AI」。

### 風箏(Kiter)——消耗戰駕駛(層1_4 起;user 2026-07-11 定打法)

1. **拉開**:與對方同排(可能被打)→ 用一張牌的移動面移開一排(不出邊界;燒牌順序=先非攻擊牌、再最弱攻擊牌)。
2. **等待**:不同排 → 不行動(收手不綁定=確立項 2026-07-11 拍——對方追齊後行動權輪回、再拉開;迫使對方燒牌追齊)。
3. **出手**:對方手牌張數 ≤2(**讀牌斷鏈**:對方連鎖上限=手牌張數)或**自身手牌 ≤2(移動燃料耗完、把殘牌打掉)** → 照鏈向規則出手。(首版曾用「手滿」當觸發=實作錯誤:補滿制下每回合開場恆手滿,風箏根本不會放——2026-07-11 修。)
4. 切入照共通框架(有牌能切就切最大傷)。
- 條件假設:**手牌張數=公開資訊**(棄牌與牌堆殘數公開的自然延伸)——報告條件欄申明。
- 定位:層1_4 風箏消耗戰的攻方模組;細節以引擎實作(l1_game.Kiter)為準。

### 回復優先(HealFirst)——回復收斂測試駕駛(層1_5 起;2026-07-12 夜間自主拍板、等 user 追認)

1. **補血**:HP 缺口 ≥ 手上最大回復牌的量、且回復每局上限未用盡 → 打出該回復牌(單張行動;缺口吃不下全額就不打,不浪費溢出)。
2. 其餘(開鏈/續打/留牌/切入)照鏈向留牌。
- 定位:「主動補血 meta」的量測代理——量回復濃度×全洗回之下對局收斂性(層1_5);不是最優回復策略(固定門檻、不看對方節奏)。
- 與鏈向留牌的差異=只有第 1 條:回復從「清手位的被動選項」升為「行動優先序第一位」。

### 格點偏移(Explorer)——exploit 掃描儀器(層2_5 起;2026-07-12 夜間自主拍板、等 user 追認)

1. 鏈向留牌的**三參數偏移**,其餘行為全同:出手門檻(最長可打段 ≥N 才開鏈;標準=2)/切入策略(必切=現行/不切/大牌才切 ≥30)/回復優先(層1_5 邏輯開關)。
2. 定位:窮舉打法空間找均衡偏移的**儀器**、不是玩法提案;實作在 `sim_v2/l25_exploit.py`。
3. ⚠️ 層2_5 主發現:**門檻1(不留牌)+不切=93.7% 輾壓留牌標準**——鏈向留牌是「設計對齊」的量測標準、**不是均衡代理**;引用電池數字帶此警語(連鎖誘因待拍=層2_5/2_5_1)。

### 鏈向留牌欄軸版(GridChain)——真池量測標準(層1_6 起;2026-07-13 自主拍板、等 user 追認)

1. **可命中才算攻擊牌**:開鏈與續打的合法集=「接得上」且「當下範圍可命中」(位移把人推走=鏈自然斷,位移卡的自懲面)。
2. **開鏈墊攻擊+5**:手上有「攻擊+5」且計畫開鏈 → 先打它(米號空過+快手窗蓋住第一個切入窗)再照鏈向邏輯打。
3. **走位**:無可命中攻擊 → 單步貪心(枚舉燃料牌×可達格,取移動後可命中火力最大);一步內無收益 → 逼近步(先對列、再壓前線;燒最弱燃料)。
4. **佔領卡當燃料**(本駕駛不主動佔領);留牌與清手位同鏈向留牌。
- 實作=`sim_v2/grid_game.py:GridChain`;風箏/回復優先/格點偏移的欄軸版照同法延伸。

### 幾何感知駕駛 stack 版（RichStack）——常備儀器（2026-07-19 建、實作裁量待追認）

1. `rich_stack.py:RichStack`＝Routed 子類＋六個可組合模組旗標（全關＝與 Routed **指紋全等**、驗收釘住＝`rich_stack_tests.py` 9 項）：避彈／讀牌／血線／位移進攻／佔領兌現／龜縮破解——行為定義同 grid 版 RichChain（本節上方）、新行動經濟適配＝開鏈與佔領走 `play_action_chip`（受一回合一次門禁）、避彈與拉開走 `do_move`（移動＝正規行動）。全開版＝`RichStack.full()`。
2. **用途**：幾何敏感結論的 A/B 複驗（首例＝[[駕駛儀器_2_0_儀器敏感度複驗]]）；**拍板級結論建議加「智對智」複驗格**（雙智世界＝另一個 meta：局長 +2–3、斬系翻身 61.8）。
3. ⚠️ **已知缺陷**：位移進攻模組＝新行動經濟下的價值陷阱（拉卡白燒使用晶片權、界線鉗制下拉不進劍程；斬系 54.4→46.1 實測）——**stack 版調參待辦**（先驗「拉完真的進程」才拉）；grid 版行為照搬、未重調參＝「會用幾何」的下限版。

### 標準量測駕駛（2026-07-18 拍）＝Routed 門檻 1

- 期望值型（γ 家族＋修正版）＝結構證明儀器、住 `l221_value.py`；不入常備。

### 佔領優先(OccupierChain)——佔領 meta 量測(層1_6)

1. 手上有佔領牌且合法(推進鎖/F 欄禁佔檢查過)→ 先佔領(單張行動);其餘全同鏈向留牌欄軸版。
- 定位:「一定佔」的下限打法、非最優運用;實作=`grid_game.py:OccupierChain`。

### 豐富化(RichChain)——六模組可組合儀器(層2_6 起;2026-07-14 批次、實作裁量待追認)

1. GridChain 加**六個可組合模組旗標**(全關=與 GridChain 對局指紋全等;每模組一條觸發驗收=grid_tests.py):
   - **避彈**:敵方投彈十字罩住我→用最弱且移動力夠的燃料走出十字(斜向=兩步制下脫離需移動力 ≥2;移動力 1 的手牌躲不掉=投彈的真實壓制面;能兼顧與對方同列就兼顧)。
   - **讀牌**:對方手牌 ≤2→出手門檻 −1(安全就搶);≥6→+1(怕被切)——公開資訊只用張數。
   - **血線**:對方 HP ≤ 我最長可打段名目傷→無視留牌直接打(斬殺);自身 HP ≤ 對方單鏈名目最大傷→先回復、無回復則拉開(每回合至多一次、防空轉;「對方單鏈名目」讀對方手牌實況=儀器口徑、非公開資訊)。
   - **位移進攻**:手有劍系(近斬/寬斬/深斬)而全不在劍程→可命中的拉卡(鉤索/竹槍)開鏈拉近;對方與我同列且手有側彈可命中→側彈開鏈推離斷他對齊。
   - **佔領兌現**:手有 K 群劍(劍/寬劍/長劍)且佔領合法→先佔領(單張行動);無劍不佔。
   - **龜縮破解**:對方連續 2 回合深欄(其後兩欄)→竹槍/投彈可命中就開鏈、否則佔領合法就佔。
2. 模組優先序(實作裁量、待追認):避彈→血線(斬殺→自保)→位移進攻→佔領兌現→龜縮破解→讀牌(門檻)+基準。
3. 實作=`grid_game.py:RichChain`;主效應與組合掃描證據=[[連鎖誘因_1_1_連鎖動機拆解]]。

### 鏈向留牌堆疊版(StackChain)——新行動經濟量測標準候選(步二起;2026-07-16 實作裁量待追認)

1. **每個行動選一件事**(新行動經濟:使用晶片一回合一次、移動每次=一個行動):
   - 本回合未用晶片且最長可命中段 ≥出手門檻 → 使用晶片(墊「攻擊+5」照舊——快手批不可切;續連照序:同碼優先、再升冪次一碼);
   - 留牌但手滿 → 清手位(受傷有回復先回復、否則打最弱單攻);留牌未滿 → 不動作;
   - 無可命中攻擊 → 單步貪心走位(移動後可命中火力最大)、無收益 → 逼近步(先對列、再壓前線;燒最弱燃料);
   - 都不成立 → 不動作。
2. **切入**:接得上且結算當下可命中的攻擊牌、挑總傷最大(必切=舊口徑沿用;非攻擊面白嫖切入=步三 exploit 掃描項)。
3. 實作=`sim_v2/stack_game.py:StackChain`;**佔領優先變體=OccupierStack**(未用晶片且佔領合法→先佔領,其餘同)。
4. 定位:對齊既有 GridChain 的行為假設搬到新行動經濟——量測標準**候選**,步三重校準時複核。

### 組牌傾向

- **一句定義**:構築牌庫時,**決定每一種晶片印哪個代碼**的分配傾向(舊名鋪碼,2026-07-11 改)。
